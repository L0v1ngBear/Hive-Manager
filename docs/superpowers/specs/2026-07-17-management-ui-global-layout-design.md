# Hive 管理端全局页面样式修复设计

日期：2026-07-17

## 目标

统一修复管理端除订单页面外，在宽桌面、866px 窄桌面和 390px 手机视口下的页面布局问题。修复必须作用于共用布局规则，并保留各业务页面针对自身信息密度的明确控制，禁止继续使用覆盖所有 `.grid` 的全局强制规则。全局规则调整不得改变订单页面当前布局。

## 范围

- 全局响应式容器、断点和表格溢出规则。
- 仪表盘、安装任务、库存、审批、员工、组织、角色、质量、客户、价格、考勤、设备、文档、公告、打印、标签和使用手册页面。
- 桌面折叠侧栏与移动端导航。
- 角色权限选择弹窗、员工个人权限抽屉和审批负责人弹窗。
- 表单可访问名称、弹窗标题关联及“蜂巢 Hive”品牌文字。

不改变订单页面、业务接口、权限码、订单状态流转、数据库结构和后端逻辑。

## 设计

### 响应式基础

保留 `ResponsivePageFrame` 的 `compact/tablet/narrow/wide` 容器模式。删除 `max-width: 900px` 下对 `.function-page-container > .grid` 的统一单列覆盖，只对明确命名且确实需要单列的布局类应用规则。

- `wide`：按页面原有 4 至 5 列统计布局。
- `narrow/tablet`：统计卡默认 2 列，业务类型卡根据内容使用 2 至 3 列。
- `compact`：单列或横向可滚动的状态控件，筛选和次要统计默认折叠。

### 共用筛选与表格

- 搜索框、下拉框和日期控件设置稳定的最小宽度，不允许收缩成箭头按钮。
- 筛选操作区保持同一行或独立操作行，按钮不孤立漂移。
- 窄桌面表格使用可见横向滚动容器，保留表头；手机才切换为卡片式行布局。
- 关键操作列保持可见，次要列通过列设置管理。

### 导航

- 桌面首次进入默认展开侧栏。
- 折叠侧栏只显示图标，使用 Tooltip 提供名称；不显示 10px 文本。
- 移动端保留当前抽屉导航，并统一品牌为“蜂巢 Hive”。

### 权限界面

- 角色权限弹窗使用按业务域分组的可搜索复选列表。
- 支持只看已选、分组全选和保存前变更摘要。
- 员工个人权限抽屉保留现有结构，提升次要文字字号和对比度。
- 弹窗通过 `aria-labelledby` 关联可见标题；筛选下拉框补充可访问名称。

## 测试策略

先添加会失败的布局契约测试，覆盖：

1. 全局样式不得在 900px 下强制所有直属 `.grid` 单列。
2. 窄桌面表格不得被全局隐藏横向溢出。
3. 全局规则调整后订单页面相关源码和视觉基线保持不变。
4. 折叠侧栏不渲染微型文字标签，并提供可访问提示。
5. 权限弹窗具有可访问名称和可浏览权限列表。
6. 品牌文字统一为“蜂巢 Hive”。

完成后执行 Node 测试、前端构建，并对宽桌面、866px 和 390px 视口进行截图回归。重点检查文字重叠、控件收缩、表格裁切、首屏内容密度和弹窗可用性。订单页面只做防回归核对，不纳入本轮样式修改。

## 验收标准

- 866px 下核心统计区不再全部变为单列。
- 价格和设备筛选控件不再收缩或裁切。
- 桌面折叠导航没有 10px 文字。
- 角色权限可搜索、分组查看和核对已选项。
- 390px 下首屏能看到主要业务内容。
- 订单页面文件没有被修改，且全局修复未造成订单页面新增回归。
- 构建与测试通过，三个代表视口无明显重叠和不可操作内容。
