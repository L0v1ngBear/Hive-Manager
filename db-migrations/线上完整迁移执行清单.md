# 线上完整迁移执行清单

以下命令默认服务器项目目录是 `/root/hive`，数据库容器名由 `docker compose` 管理。

## 1. 上传迁移体系

如果服务器已有旧的 `db-migrations`，先在服务器备份：

```bash
mkdir -p /root/hive
cd /root/hive
if [ -d db-migrations ]; then mv db-migrations db-migrations.bak.$(date +%Y%m%d_%H%M%S); fi
```

在本机 PowerShell 执行：

```powershell
$Server = "<服务器IP或域名>"
scp -r "C:\Users\HUAWEI\Desktop\hive部署_全新配置\db-migrations" "root@${Server}:/root/hive/"
```

## 2. 线上预检

```bash
cd /root/hive
bash db-migrations/scripts/preflight-online.sh
```

如果这里出现 `Got error 168`、`Unknown error from engine` 或 DDL 失败，先不要继续迁移，说明 MySQL 存储引擎或数据目录状态异常。

## 3. 备份线上库

```bash
cd /root/hive
bash db-migrations/scripts/backup-online.sh
```

备份文件会放在 `/root/hive/backups/db/时间戳/`。

## 4. 普通版本迁移

适用于线上库整体还可用，只是缺字段/缺索引：

```bash
cd /root/hive
bash db-migrations/scripts/run-versioned-migrations.sh
bash db-migrations/scripts/verify-online-schema.sh
```

## 5. 完整重建影子库

适用于线上表和数据已经混乱，需要按本地标准库完整重建，但又不想直接覆盖线上：

```bash
cd /root/hive
TARGET_DATABASE=hive_shadow RESET_TARGET=YES bash db-migrations/scripts/import-baseline-to-shadow.sh
DATABASE_NAME=hive_shadow bash db-migrations/scripts/verify-online-schema.sh
```

确认 `hive_shadow` 正常后，再决定最终切换方案。

## 6. 覆盖线上库的保护条件

默认脚本不会覆盖 `hive`。如果确实要用 baseline 覆盖线上库，必须先备份，并显式执行：

```bash
cd /root/hive
bash db-migrations/scripts/backup-online.sh
TARGET_DATABASE=hive RESET_TARGET=YES CONFIRM_IMPORT_TO_HIVE=YES bash db-migrations/scripts/import-baseline-to-shadow.sh
```

这一步会重建线上 `hive`，只有确认线上旧业务数据可以被本地标准库覆盖时才能执行。
